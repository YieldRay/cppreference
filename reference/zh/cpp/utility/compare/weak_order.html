template< class T, class U >

    requires /* 见下文 */

constexpr std::weak_ordering weak_order(T&& t, U&& u) noexcept(/* 见下文 */);

用三路比较比较两个值，并产生 std::weak_ordering 类型的结果。

设 t 与 u 为表达式，而 T 与 U 分别代表 decltype((t)) 与 decltype((u)) 类型的值，那么 std::weak_order(t, u) 表达式等价于：

  • 如果 std::is_same_v<std::decay_t<T>, std::decay_t<U>> 是 true：
    • std::weak_ordering(weak_order(t, u))，那么在不包含 std::weak_order 的声明的语境中进行重载决议的情况下它是良构的表达式，
    • 否则，如果 T 是浮点类型：
      • 如果 std::numeric_limits<T>::is_iec559 是 true，那么进行浮点值的弱序比较（见后述）并返回 std::weak_ordering 类型的值作为结果，
      • 否则，产生与 T 的比较运算符所观察的顺序一致的 std::weak_ordering 类型的值，
    • 否则是 std::weak_ordering(std::compare_three_way()(t, u))，如果它良构，
    • 否则是 std::weak_ordering(std::strong_order(t, u))，如果它良构。
  • 所有其他情况下，该表达式非良构，这能在出现于模板实例化的立即语境时导致代换失败。

定制点对象

名字 std::weak_order 代表一个定制点对象，它是某个字面 semiregular 类类型的 const 函数对象。为阐述目的，以 __weak_order_fn 表示它的类型的 cv 无限定版本。

__weak_order_fn 的所有实例均相等。在相同实参上调用 __weak_order_fn 类型的不同实例的效果是等价的，这与指代该实例的表达式是左值还是右值，以及是否为 const 限定无关（然而不要求 volatile 限定的实例是可调用的）。由此，可以自由地复制 std::weak_order 并且能彼此替代地使用它的副本。

给定类型集合 Args...，如果 std::declval<Args>()... 满足上面对于 std::weak_order 的实参的要求，那么 __weak_order_fn 就实现

  • std::invocable<__weak_order_fn, Args...>、
  • std::invocable<const __weak_order_fn, Args...>、
  • std::invocable<__weak_order_fn&, Args...> 和
  • std::invocable<const __weak_order_fn&, Args...>。

否则，__weak_order_fn 的函数调用运算符不会参与重载决议。

IEEE 浮点类型的严格弱序

设 x 和 y 为相同 IEEE 浮点类型的值，而 weak_order_less(x, y) 为指示在 C++ 标准定义的严格弱序中 x 是否前趋于 y 的布尔结果。

  • 如果 x 和 y 均非 NaN，那么当且仅当 x < y 时 weak_order_less(x, y) == true，即相等浮点值的所有表示等价；
  • 如果 x 是负 NaN 且 y 不是负 NaN，那么 weak_order_less(x, y) == true；
  • 如果 x 不是正 NaN 且 y 是正 NaN，那么 weak_order_less(x, y) == true；
  • 如果 x 和 y 都是拥有同一符号的 NaN，那么 (weak_order_less(x, y) || weak_order_less(y, x)) == false，即所有符号相同的 NaN 等价。
